# Памятка по pymavlink / mavutil
**Работа с MAVLink протоколом в Python и командной строке**

---

## 1. Что такое pymavlink и mavutil?
**pymavlink** — это библиотека на Python для работы с протоколом MAVLink (используется в дронах, наземных станциях, автопилотах: ArduPilot, PX4 и др.).  
**mavutil** — модуль внутри pymavlink, который предоставляет удобные функции для подключения, отправки и приёма сообщений, работы с параметрами, миссиями и логами.

---

## 2. Установка
```bash
pip install pymavlink
# или
pip install pymavlink --user
```
Также может потребоваться `future` и `numpy` (устанавливаются автоматически).

## 3. Основы работы в Python

### 3.1. Подключение к автопилоту
Соединение устанавливается через `mavutil.mavlink_connection()`. Поддерживаемые типы:
- последовательный порт (`/dev/ttyUSB0`, `COM3`)
- UDP (`udp:127.0.0.1:14550` — клиент, `udpin:0.0.0.0:14550` — сервер)
- TCP (`tcp:127.0.0.1:5760` — клиент, `tcpin:0.0.0.0:5760` — сервер)
- файл лога (`logfile.tlog`, `logfile.bin`)

```python
from pymavlink import mavutil

# Подключение через последовательный порт (скорость 57600)
master = mavutil.mavlink_connection('/dev/ttyUSB0', baud=57600)

# Подключение по UDP (слушаем порт 14550)
master = mavutil.mavlink_connection('udpin:0.0.0.0:14550')

# Подключение к файлу лога
master = mavutil.mavlink_connection('flight.tlog')
```

### 3.2. Ожидание heartbeat (первичная проверка связи)
После подключения обычно ждут heartbeat, чтобы убедиться, что устройство отвечает.
```python
master.wait_heartbeat()
print(f"Heartbeat from system {master.target_system}, component {master.target_component}")
```

### 3.3. Получение сообщений
Метод `recv_match()` читает следующее сообщение, можно фильтровать по типу и блокировать ожидание.
```python
# Получить любое сообщение (блокирующий режим)
msg = master.recv_match(blocking=True)

# Получить конкретное сообщение (например, GPS_RAW_INT)
msg = master.recv_match(type='GPS_RAW_INT', blocking=True)

# Получить несколько сообщений за раз
while True:
    msg = master.recv_match(type=['ATTITUDE', 'GLOBAL_POSITION_INT'], blocking=True)
    if msg.get_type() == 'ATTITUDE':
        print(f"roll: {msg.roll}, pitch: {msg.pitch}")
```

### 3.4. Отправка сообщений
**Важно:** перед отправкой команд убедитесь, что настроены `target_system` и `target_component` (обычно берутся из heartbeat).

#### Отправка HEARTBEAT (для поддержания связи, если вы — GCS)
```python
master.mav.heartbeat_send(
    mavutil.mavlink.MAV_TYPE_GCS,
    mavutil.mavlink.MAV_AUTOPILOT_INVALID,
    0, 0, 0
)
```

#### Отправка команды (MAV_CMD)
Для команд используется `mav.command_long_send()`.
```python
# Пример: ARM/DISARM
master.mav.command_long_send(
    master.target_system,
    master.target_component,
    mavutil.mavlink.MAV_CMD_COMPONENT_ARM_DISARM,
    0,  # confirmation
    1,  # param1: 1 - arm, 0 - disarm
    0, 0, 0, 0, 0, 0  # остальные параметры
)
```

#### Отправка параметра
```python
master.mav.param_set_send(
    master.target_system,
    master.target_component,
    b'PARAM_NAME',   # имя параметра в байтах
    value,
    mavutil.mavlink.MAV_PARAM_TYPE_REAL32
)
```

### 3.5. Чтение параметров
```python
# Запросить список параметров
master.mav.param_request_list_send(master.target_system, master.target_component)

# Ждать все параметры (можно собирать в словарь)
params = {}
while len(params) < 10:  # или пока не получим все
    msg = master.recv_match(type='PARAM_VALUE', blocking=True)
    params[msg.param_id] = msg.param_value
```

Удобная функция `mavutil.mavfile_param_find` или `master.params` (расширенный интерфейс):
```python
# Получить конкретный параметр
value = master.param_fetch('BATT_CAPACITY')
```

## 4. Работа с сообщениями MAVLink

### 4.1. Основные поля сообщений
Каждое сообщение имеет:
- `msg.get_type()` — тип сообщения (строка)
- `msg.to_dict()` — словарь всех полей
- Доступ к полям: `msg.roll`, `msg.lat`, `msg.alt` и т.д.

Пример:
```python
msg = master.recv_match(type='GLOBAL_POSITION_INT', blocking=True)
print(f"lat: {msg.lat / 1e7}, lon: {msg.lon / 1e7}, alt: {msg.alt / 1000}m")
```

### 4.2. Часто используемые типы сообщений
- `HEARTBEAT` — статус автопилота, режим (mode), вооружён.
- `SYS_STATUS` — напряжение, загрузка ЦП, ошибки.
- `GPS_RAW_INT` — данные GPS (координаты, количество спутников).
- `ATTITUDE` — углы ориентации (roll, pitch, yaw).
- `GLOBAL_POSITION_INT` — глобальные координаты, высота, скорость.
- `VFR_HUD` — скорость, высота, курс.
- `RC_CHANNELS` — значения каналов радиоуправления.
- `SERVO_OUTPUT_RAW` — выходы серво (моторы).
- `STATUSTEXT` — текстовые сообщения от автопилота.

### 4.3. Ожидание нескольких сообщений
```python
# Ждать два сообщения: сначала heartbeat, потом GPS
heart = master.recv_match(type='HEARTBEAT', blocking=True)
gps = master.recv_match(type='GPS_RAW_INT', blocking=True)
```

Или использовать цикл с условием:
```python
while True:
    msg = master.recv_match(type=['HEARTBEAT', 'STATUSTEXT'], blocking=True)
    if msg.get_type() == 'STATUSTEXT':
        print(f"Status: {msg.text}")
```

## 5. Работа с параметрами (удобные функции)
`master.param_fetch()` и `master.param_set()` — обёртки над MAVLink сообщениями.
```python
# Чтение
value = master.param_fetch('BATT_CAPACITY')
print(value)

# Запись
master.param_set('BATT_CAPACITY', 3000)
```

Получение списка всех параметров (блокирующий):
```python
master.param_fetch_all()
params = master.params  # словарь {имя: значение}
```

## 6. Работа с миссиями (Waypoints)

### 6.1. Чтение миссии
```python
# Запросить список миссии
master.waypoint_request_list_send()
msg = master.recv_match(type='MISSION_COUNT', blocking=True)
count = msg.count

waypoints = []
for i in range(count):
    master.waypoint_request_send(i)
    wp = master.recv_match(type='MISSION_ITEM', blocking=True)
    waypoints.append(wp)
```

### 6.2. Загрузка миссии
Сначала очистить старую, затем отправить каждую точку.
```python
master.waypoint_clear_all_send()
master.recv_match(type='MISSION_ACK', blocking=True)

# Отправка точек
for i, wp in enumerate(waypoints_list):
    master.mav.mission_item_send(
        master.target_system,
        master.target_component,
        i,          # seq
        0,          # frame (MAV_FRAME)
        mavutil.mavlink.MAV_CMD_NAV_WAYPOINT,
        0, 1,       # current, autocontinue
        0, 0, 0, 0, # params
        wp[0], wp[1], wp[2]  # x, y, z (lat, lon, alt)
    )
```

## 7. Утилиты командной строки
pymavlink включает несколько скриптов для работы с логами и параметрами. Они устанавливаются вместе с пакетом.

### 7.1. mavlogdump.py — просмотр логов
Читает TLOG или BIN файл, выводит сообщения.
```bash
mavlogdump.py --types GPS,ATTITUDE flight.tlog
```
Ключи:
- `--types` — фильтр по типам (через запятую)
- `--condition` — условие на поля (например, `GLOBAL_POSITION_INT.lat > 0`)
- `--format` — формат вывода (human, csv, mat)
- `--no-binary` — пропускать бинарные данные
- `--planner` — формат как в Mission Planner

### 7.2. mavparms.py — работа с параметрами через файл или подключение
Чтение/запись параметров автопилота или из лога.
```bash
# Чтение параметров из подключённого автопилота
mavparms.py --port /dev/ttyUSB0 --baudrate 57600 --download params.param

# Запись параметров
mavparms.py --port /dev/ttyUSB0 --baudrate 57600 --upload params.param

# Извлечь параметры из лога
mavparms.py --file flight.tlog --download params.param
```

### 7.3. mavmission.py — работа с миссиями
```bash
# Скачать миссию
mavmission.py --port /dev/ttyUSB0 --baudrate 57600 --download mission.txt

# Загрузить миссию
mavmission.py --port /dev/ttyUSB0 --baudrate 57600 --upload mission.txt
```

### 7.4. mavgraphs.py — построение графиков из лога
```bash
mavgraphs.py flight.tlog --plot "GPS_RAW_INT.lat,GPS_RAW_INT.lon"
```

### 7.5. mavftp.py — работа с файловой систему (для PX4)
```bash
mavftp.py --port udp:127.0.0.1:14550 list /
mavftp.py --port udp:127.0.0.1:14550 get /fs/microsd/logfile.bin
```

### 7.6. Другие утилиты
- `mavgen.py` — генератор кода из XML определений MAVLink (используется редко)
- `mavsigloss.py` — просмотр сигналов в логе
- `magical.py` — магические операции (продвинутая)

## 8. Пояснение основных понятий

### System ID и Component ID
- **System ID** — идентификатор всего устройства (например, автопилот = 1, GCS = 255).
- **Component ID** — идентификатор компонента внутри системы (например, автопилот = 1, камера = 100).

При отправке команд нужно указать `target_system` и `target_component` — они обычно равны тем, которые пришли в первом heartbeat.

В mavutil после `wait_heartbeat()` автоматически устанавливаются:
```python
master.target_system = msg.get_srcSystem()
master.target_component = msg.get_srcComponent()
```

### Режимы полёта (MAV_MODE)
Можно установить режим командой `set_mode`.
```python
master.mav.set_mode_send(
    master.target_system,
    mavutil.mavlink.MAV_MODE_FLAG_CUSTOM_MODE_ENABLED,
    mode_number  # номер режима (для ArduCopter: 4 = GUIDED, 0 = STABILIZE)
)
```
Или использовать константы:
```python
from pymavlink.dialects.v20 import ardupilotmega as mavlink2
mode = mavlink2.MAV_MODE_FLAG_CUSTOM_MODE_ENABLED
master.mav.set_mode_send(master.target_system, mode, 4)  # GUIDED
```

## 9. Примеры использования в консоли

### 9.1. Просмотр live-потока с автопилота
```bash
mavlogdump.py --port udp:127.0.0.1:14550 --types ATTITUDE,GPS_RAW_INT
```

### 9.2. Сохранение лога из порта
```bash
mavlogdump.py --port /dev/ttyUSB0 --baudrate 57600 --output flight.tlog
```

### 9.3. Отправка команды через командную строку (используя `mavcmd`)
В pymavlink нет отдельного скрипта `mavcmd`, но можно написать однострочник на Python:
```bash
python -c "
from pymavlink import mavutil
master = mavutil.mavlink_connection('udp:127.0.0.1:14550')
master.wait_heartbeat()
master.mav.command_long_send(
    master.target_system, master.target_component,
    mavutil.mavlink.MAV_CMD_COMPONENT_ARM_DISARM, 0, 1, 0, 0, 0, 0, 0, 0)
"
```

### 9.4. Чтение параметров через mavparms
```bash
mavparms.py --port /dev/ttyUSB0 --baudrate 57600 --list
```

### 9.5. Запись параметров
```bash
mavparms.py --port /dev/ttyUSB0 --set BATT_CAPACITY=3000
```

## 10. Советы и частые проблемы

### 10.1. Не приходит heartbeat
- Проверьте скорость (baudrate).
- Убедитесь, что порт не занят другой программой.
- Для UDP убедитесь, что вы слушаете правильный порт и отправляете с правильного адреса.

### 10.2. Ошибка `No module named 'pymavlink.dialects'`
Установите pymavlink с поддержкой диалектов (обычно по умолчанию включено). Если нет — укажите диалект явно:
```python
from pymavlink.dialects.v20 import ardupilotmega as mavutil
```

### 10.3. Как узнать номер режима для ArduPilot?
Список режимов в разных типах AP:
- **ArduCopter**: STABILIZE=0, ACRO=1, ALT_HOLD=2, AUTO=3, GUIDED=4, LOITER=5, RTL=6, CIRCLE=7, LAND=9, DRIFT=11, SPORT=13, FLIP=14, AUTOTUNE=15, POSHOLD=16, BRAKE=17, THROW=18, AVOID_ADSB=19, GUIDED_NOGPS=20, SMART_RTL=21, FLOWHOLD=22, FOLLOW=23, ZIGZAG=24, SYSTEMID=25, AUTOROTATE=26, AUTO_RTL=27, TURTLE=28.
- **ArduPlane**: MANUAL=0, CIRCLE=1, STABILIZE=2, TRAINING=3, ACRO=4, FBWA=5, FBWB=6, CRUISE=7, AUTOTUNE=8, AUTO=10, RTL=11, LOITER=12, AVOID_ADSB=13, GUIDED=15, INITIALISING=16, QSTABILIZE=17, QHOVER=18, QLOITER=19, QLAND=20, QRTL=21, QAUTOTUNE=22, QACRO=23, THERMAL=24, LOITER_TO_QLAND=25.

Для PX4 используются базовые режимы из стандарта MAVLink.

### 10.4. Как отправить произвольное сообщение?
Используйте метод `master.mav.<message_name>_send(...)`. Список сообщений доступен через `dir(master.mav)`.

### 10.5. Как проверить, вооружён ли автопилот?
Из heartbeat: бит `base_mode & MAV_MODE_FLAG_SAFETY_ARMED`.
```python
msg = master.recv_match(type='HEARTBEAT', blocking=True)
armed = (msg.base_mode & mavutil.mavlink.MAV_MODE_FLAG_SAFETY_ARMED) != 0
```

### 10.6. Как отправить RC оверрайд?
```python
master.mav.rc_channels_override_send(
    master.target_system, master.target_component,
    1500, 1500, 1500, 1500,  # каналы 1-4
    0, 0, 0, 0               # каналы 5-8
)
```

# Программа управления мультироторным БПЛА (Ardupilot SITL)

Проект реализует полетную программу для симулятора Ardupilot SITL с использованием протокола MAVLink и библиотеки `pymavlink`. Программа позволяет подключиться к дрону, выполнить взлет, полет по относительным координатам, возврат домой и посадку. Код написан на Python, а геодезические расчеты вынесены в отдельный класс, реализующий преобразования по ГОСТ 32453-2017.

---

## Реализованные классы

### 1. `Multirotor` (файл `Multirotor.py`)

Основной класс, инкапсулирующий все операции по управлению дроном через MAVLink. Он хранит текущее состояние (координаты, домашнюю точку), управляет подключением и отправкой команд.

#### **Инициализация и подключение**
```python
drone = Multirotor(connection_string='tcp:127.0.0.1:5762')
```
- При создании объекта устанавливается соединение с указанным адресом.
- Ожидается **heartbeat** (таймаут 3 секунды), иначе выбрасывается `ConnectionError`.
- Запрашивается поток телеметрии (`MAV_DATA_STREAM_ALL`).
- Вызывается `_save_home_position()`, которая получает текущую позицию и сохраняет её как домашнюю.

#### **Внутренние методы для работы с позицией**
- `_update_current_position()` – неблокирующее чтение сообщения `GLOBAL_POSITION_INT` и обновление атрибутов `lat`, `lon`, `alt`.
- `_get_current_position(timeout)` – блокирующее получение позиции с повторными попытками в течение заданного времени.
- `_calculate_distance_to_home()` – вычисляет расстояние до дома, используя `_get_current_position` и функцию `_calculate_distance_between_points` (которая обращается к `TranslationWGS84.calculate_distance_in_local_tangent`).

#### **Основные публичные методы**
| Метод | Описание |
|-------|----------|
| `set_mode(mode)` | Устанавливает режим полета (`GUIDED`, `AUTO`, `RTL`, `LAND`). Проверяет подтверждение смены режима по heartbeat. |
| `arm()` | Армирует двигатели (`arducopter_arm()`) и ожидает подтверждения. |
| `takeoff(altitude)` | Отправляет команду взлета на заданную высоту. Ожидает достижения 90% от цели (таймаут 60 с). |
| `flight_to(dx, dy, target_altitude=0, target_radius=0.5)` | Выполняет полет в точку, смещенную относительно текущей позиции на `dx` (восток) и `dy` (север).<br>• Если `target_altitude == 0`, сохраняется текущая высота.<br>• Преобразование смещений в глобальные координаты выполняется через `TranslationWGS84.convert_local_tangent_to_geodetic`.<br>• Во время полета каждые 0.1 с выводится текущее расстояние до цели. При достижении половины пути выводится сообщение.<br>• Цель считается достигнутой, когда расстояние становится меньше `target_radius` в течение 5 циклов (для стабильности). |
| `home()` | Возвращает дрон в домашнюю точку на **текущей высоте**, затем автоматически вызывает `land()`. Если расстояние по горизонтали уже меньше `HOME_RADIUS`, сразу переходит к посадке. Также отслеживает половину пути. |
| `land()` | Инициирует посадку в текущей позиции. Контролирует снижение; при зависании на малой высоте (>20 циклов без изменения) принудительно отключает двигатели. |
| `disarm()` | Отключает двигатели. |
| `get_status()` | Возвращает словарь с текущей позицией, домашней позицией и расстоянием до дома. |

#### **Обработка ошибок и крайних случаев**
- Все методы обернуты в `try-except`, при ошибках выводится сообщение и возвращается `False`.
- При получении позиции используются таймауты (`POSITION_TIMEOUT`), чтобы избежать бесконечного ожидания.
- Во время полета обрабатывается `KeyboardInterrupt` – программа завершается корректно.

#### **Использование геодезических преобразований**
`Multirotor` активно использует статические методы класса `TranslationWGS84`:
- Для расчета расстояния между двумя точками (`_calculate_distance_between_points` вызывает `calculate_distance_in_local_tangent`).
- Для преобразования локальных смещений (dx, dy) в глобальные координаты (`convert_local_tangent_to_geodetic`).

---

### 2. `TranslationWGS84` (файл `TranslationWGS84.py`)

Класс-утилита, реализующий преобразования координат в соответствии с **ГОСТ 32453-2017**. Содержит только статические методы и не хранит состояние. Отвечает за:
- Пересчет между геодезическими (широта, долгота, высота) и геоцентрическими прямоугольными координатами (X, Y, Z) для различных датумов (WGS-84, ГСК-2011, ПЗ-90.11, СК-42/95).
- Преобразование между датумами (например, ПЗ-90.11 ↔ WGS-84, ПЗ-90.11 ↔ ГСК-2011, ПЗ-90.11 ↔ СК).
- Проекцию Гаусса-Крюгера (для эллипсоида Красовского) – методы `convert_to_gauss_krueger` и `convert_from_gauss_krueger`.
- Работу с локальной касательной плоскостью (ENU): `convert_geodetic_to_local_tangent`, `convert_local_tangent_to_geodetic`.
- Вычисление расстояния и азимута в локальной плоскости.

#### **Ключевые методы, используемые в Multirotor**
- `calculate_distance_in_local_tangent(point1, point2)` – возвращает евклидово расстояние между двумя точками в горизонтальной плоскости, принимая первую точку за начало локальных координат.
- `convert_local_tangent_to_geodetic(east, north, up, origin_lat, origin_lon, origin_alt)` – преобразует локальные смещения (восток, север, вверх) обратно в глобальные геодезические координаты WGS-84.

#### **Особенности реализации**
- Все углы задаются в **радианах** (кроме методов Гаусса-Крюгера, где долгота может быть в градусах для определения зоны).
- Для преобразований между датумами используются параметры из ГОСТ, реализованы прямые и обратные формулы.
- Итеративный алгоритм перехода от декартовых к геодезическим координатам обеспечивает высокую точность (допуск `eps = 1e-10`).
- Класс написан так, чтобы его можно было легко расширять новыми датумами (добавление параметров в словарь `_ELLIPSOID_PARAMS` и соответствующих функций преобразования).

## Пример использования (файл `main.py`)

```python
from Multirotor import Multirotor

# 1. Подключение
drone = Multirotor()

# 2. Проверка статуса
print(drone.get_status())

# 3. Подготовка к полету
drone.set_mode('GUIDED')
drone.arm()

# 4. Взлет на 7 метров
drone.takeoff(7)

# 5. Полет по прямоугольному маршруту
drone.flight_to(100, 300)   # северо-восток
drone.flight_to(-100, 0)     # запад

# 6. Возврат домой и посадка
drone.home()
```

В консоль выводятся все этапы: текущее расстояние до цели, сообщение о половине пути, достижение цели и т.д.
